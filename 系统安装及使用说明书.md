## 第一部分：系统安装步骤

### 1. 环境要求

在安装本系统之前，请确保您的计算机已安装以下软件：

| 软件名称 | 版本要求 | 说明 |
|----------|----------|------|
| JDK | 16 或更高版本 | Java开发环境 |
| MySQL | 8.0 或更高版本 | 数据库服务 |
| Node.js | 16.0 或更高版本 | JavaScript运行环境 |
| pnpm | 8.0 或更高版本 | 前端包管理工具 |
| Maven | 3.6+ | Java项目构建工具（可选） |

#### 环境检查命令

```bash
# 检查Java版本
java -version

# 检查MySQL版本
mysql --version

# 检查Node.js版本
node -v

# 检查pnpm版本
pnpm -v
```

### 2. 数据库安装与配置

#### 2.1 安装MySQL数据库

如果尚未安装MySQL，请前往 [MySQL官网](https://dev.mysql.com/downloads/mysql/) 下载并安装MySQL 8.0版本。

#### 2.2 创建数据库

1. 启动MySQL服务
2. 使用MySQL命令行或图形化工具（如Navicat、MySQL Workbench）执行以下操作：

```sql
-- 创建数据库
CREATE DATABASE competition_platform DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

#### 2.3 导入数据库脚本

数据库脚本位于项目根目录的 `sql/competition_platform.sql` 文件中。

**方式一：命令行导入**

```bash
# 在项目根目录下执行
mysql -u root -p competition_platform < sql/competition_platform.sql
```

**方式二：MySQL命令行内导入**

```bash
# 登录MySQL
mysql -u root -p

# 切换到目标数据库
USE competition_platform;

# 执行SQL文件
SOURCE D:/UniversityCompetitionManagementSystem/sql/competition_platform.sql;
```

**方式三：图形化工具导入**

使用Navicat或MySQL Workbench等工具，连接数据库后执行 `sql/competition_platform.sql` 文件。

#### 2.4 验证数据库

导入成功后，数据库应包含以下数据表：

```
user                    # 用户表
student_detail          # 学生详细信息表
teacher_detail          # 教师详细信息表
competition             # 竞赛表
competition_registration # 竞赛报名表
team                    # 团队表
team_member             # 团队成员关系表
team_recruitment        # 团队招募需求表
guidance_relation       # 师生指导关系表
notification            # 通知消息表
announcement            # 系统公告表
```

### 3. 后端安装与配置

#### 3.1 进入后端目录

```bash
cd back/competition-platform-backend
```

#### 3.2 修改配置文件

打开 `src/main/resources/application.yml` 文件，根据实际情况修改以下配置：

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/competition_platform?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root          # 修改为您的MySQL用户名
    password: mysql         # 修改为您的MySQL密码
```

**配置项说明：**

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `server.port` | 8080 | 后端服务端口 |
| `spring.datasource.url` | localhost:3306 | 数据库连接地址 |
| `spring.datasource.username` | root | 数据库用户名 |
| `spring.datasource.password` | mysql | 数据库密码 |

#### 3.3 安装依赖并启动

**方式一：使用Maven直接运行**

```bash
mvn spring-boot:run
```

**方式二：打包后运行**

```bash
# 打包项目
mvn clean package -DskipTests

# 运行JAR包
java -jar target/competition-platform-1.0.0.jar
```

#### 3.4 验证后端启动

后端启动成功后，控制台将显示：

```
Started CompetitionApplication in X.XX seconds
```

访问 `http://localhost:8080/api` 测试接口是否正常响应。

### 4. 前端安装与配置

#### 4.1 进入前端目录

```bash
cd front/competition-platform-frontend
```

#### 4.2 安装依赖

```bash
pnpm install
```

如果安装速度较慢，可以使用国内镜像：

```bash
pnpm config set registry https://registry.npmmirror.com
pnpm install
```

#### 4.3 配置后端代理（可选）

如需修改后端接口地址，编辑 `vite.config.js` 文件：

```javascript
server: {
  port: 8081,
  proxy: {
    '/api': {
      target: 'http://localhost:8080',  // 修改为后端实际地址
      changeOrigin: true
    }
  }
}
```

#### 4.4 启动前端开发服务器

```bash
pnpm run dev
```

启动成功后，终端将显示：

```
  VITE v4.4.9  ready in XXX ms

  ➜  Local:   http://localhost:8081/
  ➜  Network: http://192.168.X.X:8081/
```

## 第二部分：系统使用说明书

### 1. 系统概述

#### 1.1 系统简介

高校竞赛管理系统是一个面向高等院校的竞赛管理平台，旨在为学校师生提供便捷的竞赛信息发布、团队组建、指导申请等服务。系统采用前后端分离架构，前端使用Vue3框架，后端使用Spring Boot框架，具有良好的用户体验和扩展性。

#### 1.2 技术架构

- **前端**：Vue 3 + Vue Router + Pinia + Element Plus + Vite
- **后端**：Spring Boot 2.7 + MyBatis-Plus + MySQL 8.0
- **认证**：JWT Token 无状态认证

#### 1.3 系统特点

- 多角色权限管理（管理员、教师、学生）
- 竞赛全生命周期管理
- 灵活的团队组建机制
- 教师指导关系管理
- 站内通知消息系统
- 富文本公告发布

---

### 2. 用户角色与权限

系统支持三种用户角色，各角色权限如下：

| 角色 | 角色值 | 主要权限 |
|------|--------|----------|
| **管理员** | 0 | 用户管理、竞赛管理、团队管理、公告管理、系统统计 |
| **教师** | 1 | 发布竞赛、查看团队、接受/拒绝指导申请、管理个人指导状态 |
| **学生** | 2 | 查看竞赛、报名参赛、创建/加入团队、申请教师指导 |

### 3. 功能模块说明

#### 3.1 用户管理模块

##### 3.1.1 用户登录

1. 访问系统首页，点击右上角【登录】按钮
2. 输入用户名和密码
3. 输入验证码（如系统启用）
4. 点击【登录】按钮完成登录

##### 3.1.2 用户注册

**学生注册：**
1. 点击登录页面的【学生注册】链接
2. 填写用户名、密码、姓名、学号、学院、专业、年级等信息
3. 点击【注册】按钮完成注册

**教师注册：**
1. 点击登录页面的【教师注册】链接
2. 填写用户名、密码、姓名、工号、学院、职称、研究方向等信息
3. 点击【注册】按钮完成注册

##### 3.1.3 个人中心

登录后点击右上角用户头像，选择【个人中心】可进行：
- 查看和修改个人信息
- 修改登录密码
- 查看个人相关数据统计

---

#### 3.2 竞赛管理模块

##### 3.2.1 查看竞赛列表

1. 点击导航栏【竞赛】菜单
2. 可按竞赛状态筛选：未开始、报名中、进行中、已结束
3. 支持按竞赛名称搜索
4. 点击竞赛卡片可查看详情

##### 3.2.2 竞赛详情

竞赛详情页面包含：
- 竞赛基本信息（名称、时间、地点、主办方）
- 竞赛介绍（富文本内容）
- 报名要求（团队人数限制等）
- 报名按钮（报名期间可见）

##### 3.2.3 发布竞赛（管理员/教师）

1. 进入竞赛管理页面
2. 点击【发布竞赛】按钮
3. 填写竞赛信息：
   - 竞赛名称
   - 竞赛时间
   - 报名时间
   - 竞赛地点
   - 主办方
   - 团队人数要求（最小/最大人数）
   - 竞赛介绍（富文本编辑）
4. 点击【发布】完成创建

##### 3.2.4 竞赛报名（学生）

1. 进入竞赛详情页面
2. 确认已创建或加入团队
3. 点击【立即报名】按钮
4. 选择参赛团队
5. 确认报名信息后提交

---

#### 3.3 团队管理模块

##### 3.3.1 查看团队列表

1. 点击导航栏【团队】菜单
2. 查看所有团队信息
3. 支持按团队状态筛选（招募中/已满员）
4. 支持按团队名称搜索

##### 3.3.2 创建团队

1. 进入团队页面，点击【创建团队】按钮
2. 填写团队信息：
   - 团队名称
   - 团队描述
   - 招募需求（可选）
3. 点击【创建】完成团队创建
4. 创建者自动成为团队队长

##### 3.3.3 加入团队

**方式一：申请加入**
1. 在团队列表中找到目标团队
2. 点击【申请加入】按钮
3. 等待队长审批

**方式二：通过招募信息加入**
1. 查看团队的招募需求
2. 点击【申请加入】提交申请

##### 3.3.4 团队管理（队长）

队长可在团队详情页进行以下操作：
- 查看成员列表
- 审批加入申请
- 移除成员
- 转让队长
- 发布招募需求
- 解散团队

##### 3.3.5 退出团队

普通成员可在团队详情页点击【退出团队】按钮退出当前团队。

---

#### 3.4 指导关系模块

##### 3.4.1 查看教师列表

1. 点击导航栏【指导】菜单
2. 查看所有教师的指导状态
3. 支持按研究方向筛选教师
4. 支持按教师姓名搜索

##### 3.4.2 申请指导（学生）

1. 在教师列表中找到目标教师
2. 确认教师指导状态为"接受指导"
3. 点击【申请指导】按钮
4. 填写申请说明
5. 提交申请等待教师审批

##### 3.4.3 处理指导申请（教师）

1. 进入指导管理页面
2. 查看待处理的指导申请
3. 点击【同意】或【拒绝】处理申请

##### 3.4.4 管理指导关系

- 教师/学生可查看当前的指导关系列表
- 可结束已有的指导关系

---

#### 3.5 公告通知模块

##### 3.5.1 查看系统公告

1. 点击导航栏【公告】菜单
2. 查看所有系统公告
3. 点击公告可查看详情

##### 3.5.2 发布公告（管理员）

1. 进入公告管理页面
2. 点击【发布公告】按钮
3. 填写公告标题和内容
4. 点击【发布】完成发布

##### 3.5.3 站内通知

- 用户登录后可在右上角看到通知图标
- 点击可查看所有通知消息
- 系统会在以下情况发送通知：
  - 团队申请被审批
  - 指导申请被审批
  - 竞赛报名成功
  - 系统公告

### 4. 常见问题

#### Q1: 忘记密码怎么办？

请联系系统管理员重置密码，或使用管理员账号登录后在用户管理中修改。

#### Q2: 无法连接数据库？

请检查：
1. MySQL服务是否已启动
2. 数据库连接配置是否正确
3. 数据库用户名密码是否正确
4. 防火墙是否开放3306端口

#### Q3: 前端页面无法访问后端接口？

请检查：
1. 后端服务是否已启动
2. 前端代理配置是否正确
3. 是否存在跨域问题（后端已配置CORS）

#### Q4: 登录后Token过期怎么办？

Token默认有效期为7天，过期后请重新登录。

#### Q5: 如何修改系统端口？

- 后端：修改 `application.yml` 中的 `server.port`
- 前端：修改 `vite.config.js` 中的 `server.port`
